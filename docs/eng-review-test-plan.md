# Test Plan

Generated by the engineering review on 2026-07-19 and extended for P1 on 2026-07-20. Primary P1 source: `.planning/phases/02-p1-practice-harmony/02-01-PLAN.md`.

## Affected Pages/Routes

- `/` TUNING — verify the V1 instrument shell, default Light state, mic lifecycle, engine switch, pitch display, and keyboard.
- `/` PRACTICE — verify lazy loading, key-aware harmony, chord audition, target grading, and the shared piano/audio session.

## Key Interactions to Verify

- The first load does not request microphone permission or any Neural asset.
- `MIC START` moves through permission and capture states; `STOP` releases every track and worker.
- Selecting `NEURAL` shows transfer disclosure, staged loading, cancel/retry, and an atomic live switch only after warm-up.
- Neural failure or cancellation leaves a live Light detector and an understandable reason.
- Pointer, computer-keyboard, and multi-touch input hold and release up to six reference tones independently while grading stays gated.
- Keyboard-only octave `−` / `+` shortcuts step through C2-B4, C3-B5, and C4-B6 without wrapping or overlapping keys; the status pad shows the current range without clickable octave buttons.
- The shared pitch-modulation wheel bends every sounding voice, switches between ±2 and ±12 semitones, supports global `↑` / `↓` control, and springs back to center.
- TUNING and PRACTICE switch by pointer and arrow keys without recreating a running microphone session.
- PRACTICE renders color/tension, diatonic-core, and related/borrowed lanes with key-correct spellings.
- Root, arpeggio, and chord audition use the bounded voice bank and pause grading through the 300 ms release tail.
- Selecting a chord tone clears stale results and requires a fresh 600 ms voiced window.

## Edge Cases

- Permission denied, missing device, prompt left pending, track ended, tab suspend/resume, and repeated rapid start/stop.
- Silence, input below -55 dBFS, clipping, unstable pitch, vibrato, glissando, and note-boundary hysteresis.
- Neural offline first load, abort during model fetch, model checksum mismatch, tensor-name drift, warm-up timeout, and inference backlog.
- 320 px mobile width, 200% zoom, reduced motion, keyboard-only operation, and color-independent tuning labels.
- Multi-key rollover, independent note release, pitch-modulation center return, and octave limits.
- Corrupt or blocked local storage, failed Practice chunk loading, rapid chord replay, and mode changes during reference playback.
- F major B♭, F♯ major E♯, E♭ minor G♭/C♭, and A minor harmonic-dominant G♯ spellings.

## Critical Paths

1. Open app → confirm Light default and zero neural requests → start mic → receive a stable note → stop cleanly.
2. Start Light → select Neural → load model → receive SwiftF0 frames → switch back to Light without restarting mic.
3. Start Light → select Neural → cancel or force a load error → confirm Light keeps producing frames.
4. Play A4 → confirm 440 Hz oscillator and `REFERENCE PLAYING` → release → detector resumes after fixed tail.
5. Build production output → measure initial and Neural graphs separately → fail on any hard-cap or unexpected backend asset.
6. Open TUNING → confirm zero Practice requests → select PRACTICE → confirm the Practice JS/CSS graph loads once.
7. Start the mic → switch to PRACTICE → select A minor/A4 → reach LOCKED → return to TUNING with the same live mic.
8. Select a chord → play CHORD → confirm grading is gated → confirm the gate closes after the release tail with no stuck voice.
9. At 320 px → open PRACTICE → confirm document has no horizontal overflow while chord lanes and piano scroll independently.
10. Hold A+S+D → release S → confirm C4 and E4 continue → bend ±2 ST → release the wheel → confirm exact center.
